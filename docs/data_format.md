# Data format specification

ChromaGraphNet expects a window-aligned representation of all input modalities. Each window covers 4.01 Mb of genomic sequence centered on a target locus, divided into bins at three resolutions:

| Resolution | Purpose              | Bin count per window |
|------------|----------------------|----------------------|
| 50 bp      | Conv1D inputs        | 80,200               |
| 500 bp     | Co-accessibility     | 8,020                |
| 10 kb      | Anchor / output bins | 401                  |

## Required tensors

### `acc_ctcf` (required)

```
shape: (2, 80_200)
dtype: float32
channel 0: pseudobulk scATAC accessibility (library-size normalized, log1p)
channel 1: CTCF signal (FIMO motif max score per bin, or CTCF ChIP coverage)
```

Generation: aggregate scATAC fragments into 50 bp bins, divide by total library size, take log1p. CTCF channel from `motifmatchr` against JASPAR PWMs or from CTCF ChIP-seq BigWig.

### `coacc` (required)

```
shape: (40, 8_020)
dtype: float32
content: V-stripe slice of the 500 bp Jaccard co-accessibility matrix
```

Generation:
1. Run ArchR to identify metacells from scATAC.
2. Compute pairwise Jaccard similarity between 500 bp peaks across metacells.
3. Extract the V-stripe: the 40 anchor bins (corresponding to the central 20 kb) crossed with the 8,020 flanking bins (full +/- 2 Mb at 500 bp).

The V-stripe is shaped to match the receptive field of the model.

## Optional tensors

### `rna`

```
shape: (401, rna_in_dim)               default rna_in_dim = 1
dtype: float32
content: pseudobulk gene expression aggregated to 10 kb anchor bins
```

Generation: from scRNA-seq counts, compute log1p(CP10K) per gene, then sum or mean over the genes overlapping each 10 kb bin. Multi-channel input (e.g., separate nascent vs. mature transcripts) is supported.

### `chip`

```
shape: (n_marks, 80_200)               default n_marks = 5
dtype: float32
content: log1p coverage at 50 bp bins for each histone mark
```

Default mark order: H3K4me1, H3K4me3, H3K27ac, H3K27me3, H3K36me3.

Generation: read BigWig files with pyBigWig at 50 bp resolution, take log1p. Adjust `chip_n_marks` in the config if using a different panel.

### `motif`

```
shape: (401, n_factors)                default n_factors = 200
dtype: float32
content: max FIMO motif score per factor in each 10 kb anchor bin
```

Default panel: top 200 factors from JASPAR core vertebrates.

Generation: scan the reference genome with FIMO using the JASPAR panel, then bin to 10 kb taking the max score per (bin, factor) pair.

## Optional graph priors

### `coacc_matrix`

```
shape: (401, 401)
dtype: float32
content: pairwise co-accessibility at 10 kb bin resolution
```

Used to construct co-accessibility edges. Generated by averaging the 500 bp co-accessibility within each 10 kb x 10 kb block.

### `hic_prior`

```
shape: (401, 401)
dtype: float32
content: HiC-DC+ Z-score (or similar contact-strength score) at 10 kb
```

Used to construct Hi-C-prior edges with FDR-thresholded significant interactions.

### `ctcf_orientation`

```
shape: (401,)
dtype: int (or long; values in {-1, 0, +1})
content: dominant CTCF motif orientation per anchor bin
         +1 = forward, -1 = reverse, 0 = no CTCF motif above threshold
```

Used to flag convergent vs. non-convergent CTCF pairs in edge features.

## NPZ packaging

For inference via `chromagraphnet-predict`, package all available tensors into a single NPZ file:

```python
import numpy as np

np.savez_compressed(
    "window_chr19_4.0Mb.npz",
    acc_ctcf=acc_ctcf.astype(np.float32),
    coacc=coacc.astype(np.float32),
    rna=rna.astype(np.float32),
    chip=chip.astype(np.float32),
    motif=motif.astype(np.float32),
    coacc_matrix=coacc_matrix.astype(np.float32),
    hic_prior=hic_prior.astype(np.float32),
    ctcf_orientation=ctcf_orientation.astype(np.int8),
)
```

Then run:

```bash
chromagraphnet-predict \
    --checkpoint checkpoints/chromagraphnet-base-v0.1.pt \
    --input window_chr19_4.0Mb.npz \
    --output predictions.npz
```

## Reference preprocessing pipelines

Reference preprocessing pipelines are in preparation:

- `scripts/preprocess/atac_to_window.py`: scATAC fragments to acc_ctcf + coacc
- `scripts/preprocess/rna_to_window.py`: scRNA-seq to per-bin expression
- `scripts/preprocess/chip_to_window.py`: BigWig to ChIP tensor
- `scripts/preprocess/motifs_to_window.py`: FIMO scans to motif tensor

Until those ship, see the [ChromaFold preprocessing pipeline](https://github.com/viannegao/ChromaFold/tree/main/preprocessing_pipeline) for the accessibility branches; the other modalities use standard tools.
